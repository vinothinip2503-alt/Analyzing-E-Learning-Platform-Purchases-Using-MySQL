# 📚 E-Learning Platform Analysis (MySQL Project)

## 📌 Overview

This project analyzes purchase behavior on an e-learning platform using **MySQL**. It focuses on understanding learner activity, course performance, and revenue generation through structured queries and relational database design.

---

## 🎯 Objectives

* Analyze how learners purchase courses
* Identify top-performing courses and categories
* Evaluate learner spending behavior
* Compare revenue across countries
* Detect inactive learners and underperforming courses

---

## 🛠️ Tools & Technologies

* MySQL
* SQL (Joins, Aggregations, Subqueries)
* Relational Database Design

---

## 🗄️ Database Structure

The project uses a database named `elearning_db` with three tables:

### 1. learners

* Stores learner details (ID, name, country)

### 2. courses

* Contains course information (name, category, price)

### 3. purchases

* Records transactions with foreign key relationships

---

## 🔗 Key SQL Concepts Used

* INNER JOIN, LEFT JOIN, RIGHT JOIN
* GROUP BY & Aggregation (SUM, COUNT)
* Subqueries
* Filtering using HAVING
* Sorting and ranking (ORDER BY, LIMIT)

---

## 📊 Key Analysis Performed

* 📌 Total spending per learner
* 📌 Top 3 most purchased courses
* 📌 Category-wise revenue analysis
* 📌 Multi-category learners identification
* 📌 Courses with no purchases
* 📌 Country-wise revenue comparison
* 📌 High-value learners (above average spending)
* 📌 Purchase frequency per learner

---

## 📈 Key Insights

* Data Analytics courses generate the highest revenue
* SQL and Python are the most popular courses
* Some learners purchase across multiple categories
* One learner is completely inactive
* Excel course has no purchases (low demand)
* India and Australia contribute significant revenue
